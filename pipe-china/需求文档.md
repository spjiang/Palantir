# 管网单点运维需求说明（管道泄漏风险识别与处置）

## 1. 运维点

- 运维点名称：管道泄漏风险识别与处置

## 2. 对象（Object）清单

- 管段（PipelineSegment）
- 传感器（Sensor）：压力、流量
- 告警（Alarm）：压力异常、流量异常
- 风险状态（RiskState）：正常、疑似泄漏、确认泄漏
- 运维任务（MaintenanceTask）：巡检、处置
- 证据（Evidence）：定位、照片
- 事件（Incident）

## 3. 挂载关系

- 管段 has_sensor 传感器
- 告警 related_to 管段
- 管段 has_risk_state 风险状态
- 运维任务 targets 管段
- 运维任务 has_evidence 证据
- 事件 contains 告警、运维任务、证据

## 4. 行为（Behavior）定义

约束：每个行为必须至少挂 1 个对象（建议挂到“管段”）。

### 4.1 异常识别（DetectAnomaly）

- 作用对象：管段
- 触发条件：压力异常 或 流量异常
- 状态迁移：正常 → 疑似泄漏
- 产出：异常识别记录（Artifact）

### 4.2 处置决策（DecideResponseAction）

- 作用对象：管段
- 触发条件：风险状态=疑似泄漏
- 产出：运维任务=巡检（MaintenanceTask）

### 4.3 执行与回写（ExecuteAndWriteBack）

- 作用对象：运维任务（关联管段）
- 执行要求：证据=定位 + 照片
- 状态迁移：
  - 证据充分：疑似泄漏 → 确认泄漏
  - 证据不足：疑似泄漏 → 正常
